# ScoutSystem 角色權限記錄

> 本文件用於記錄目前討論的角色權限，避免開發過程中混淆。

---

## 目前策略（2026-06-13）

**階段性權限開放策略：**

1. **第一階段**：先讓 `super_admin` 擁有**所有功能卡片**（一次過處理完）。
2. **第二階段**：逐步處理其他角色的權限（admin、group_leader、branch_leader、coach、parent、member）。
3. **第三階段**：最後才把 `super_admin` 的權限收斂到最終定義。

---

## 最終權限定義（目標）

| 角色            | 主要入口   | 可使用的功能                                                                 | 備註 |
|-----------------|------------|-------------------------------------------------------------------------------|------|
| **super_admin** | `/admin`   | 擁有所有功能（支部管理、成員資料庫、家長審核、活動管理、圖書館標記、通告管理、系統設定等） | 最高權限 |
| **admin**       | `/admin`   | 管理所有支部、審核家長申請、活動管理、圖書館標記、通告管理                    | 接近 super_admin，但不能建立管理員 |
| **group_leader**| `/leader`  | 管理**所屬支部**的活動、家長審核、成員資料、圖書館標記                        | 只能管理自己支部 |
| **branch_leader**| `/leader` | 管理**所屬支部**的活動、家長審核、成員資料、圖書館標記                        | 權限低於團長 |
| **coach**       | `/leader`  | 管理**所屬支部**的活動、圖書館標記                                            | 無審核權限 |
| **parent**      | `/parent`  | 管理子女資料、回覆活動                                                        | - |
| **member**      | `/member`  | 查看/管理自己資料（僅童軍或以上支部）                                         | - |

## 目前開發順序建議

1. 先把 `super_admin` 的所有功能卡片接好（包含 `/admin`、`活動管理`、`圖書館標記`、`通告管理` 等）
2. 之後再逐一處理其他角色
3. 最後才把 `super_admin` 權限收斂
